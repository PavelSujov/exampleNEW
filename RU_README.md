# Интерактивная база данных для анализа дисков резки полупроводниковых пластин

## Обзор приложения

Данное приложение на основе Streamlit предоставляет интерактивный интерфейс для анализа и визуализации данных, связанных с резкой полупроводниковых пластин с использованием прецизионных режущих дисков. Приложение позволяет инженерам и техникам анализировать параметры резки, оптимизировать процессы и принимать решения на основе данных для повышения эффективности производства.

### Онлайн-приложение
Доступ к онлайн-приложению: [https://examplenew.streamlit.app/](https://examplenew.streamlit.app/)

### Функциональные возможности
- **Фильтрация данных**: Интерактивные фильтры для материалов, типов резки, диапазонов толщины и ширины реза
- **Визуальная аналитика**: Интерактивные графики, показывающие взаимосвязи между параметрами резки и показателями производительности
- **Декодирование артикулов**: Инструмент для декодирования номеров артикулов дисков и получения информации о конкретных параметрах
- **Анализ производительности**: Метрики для сколов, показателей производительности и срока службы дисков
- **Оптимизация параметров**: Рекомендации по оптимальным параметрам резки на основе свойств материала
- **Загрузка пользовательской базы данных**: Пользователи могут загружать свои собственные файлы базы данных для анализа

### Технологическое значение
Приложение использует современные методы науки о данных для преобразования исходных данных о процессе резки в действенные выводы. Используя pandas для манипуляции данными и Plotly для интерактивных визуализаций, оно предоставляет сложную аналитическую платформу, которая позволяет инженерам выявлять закономерности, корреляции и возможности оптимизации в процессе резки пластин.

### Экономическое влияние
- **Оптимизация процессов**: Снижает отходы материалов за счет выявления оптимальных параметров резки
- **Эффективность оборудования**: Увеличивает срок службы дисков за счет лучшего выбора параметров
- **Контроль качества**: Минимизирует сколы и дефекты, снижая процент брака
- **Снижение затрат**: Повышает общую эффективность производства и снижает эксплуатационные расходы
- **Поддержка принятия решений**: Обеспечивает принятие решений на основе данных для улучшения процессов

### Разработка и развертывание
- **Поддержка Docker**: Контейнеризованное приложение для согласованного развертывания
- **GitHub Actions**: Автоматизированный конвейер тестирования и развертывания
- **Покрытие тестами**: Комплексные модульные тесты, обеспечивающие качество кода
- **Качество кода**: Следует стандартам PEP 8 с аннотациями типов и документацией

## Установка

### Необходимые компоненты
- Python 3.14+
- менеджер пакетов pip

### Настройка
1. Клонируйте репозиторий:
   ```bash
   git clone <url-адрес-репозитория>
   cd exampleNEW
   ```

2. Создайте и активируйте виртуальное окружение:
   ```bash
   python -m venv venv
   venv\Scripts\activate  # В Windows
   source venv/bin/activate  # В macOS/Linux
   ```

3. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```

4. Запустите приложение:
   ```bash
   streamlit run app.py
   ```

## Использование

### Локальная разработка
1. Активируйте виртуальное окружение
2. Запустите `streamlit run app.py`
3. Откройте приложение по адресу `http://localhost:8501`

### Docker-развертывание
1. Соберите образ:
   ```bash
   docker build -t streamlit-disc-analysis-app .
   ```

2. Запустите контейнер:
   ```bash
   docker run -p 8501:8501 streamlit-disc-analysis-app
   ```

## Развертывание

### Конвейер GitHub Actions
Проект включает в себя автоматизированный CI/CD конвейер, который:
1. Запускает тесты при каждом пуше и запросе на слияние
2. Собирает и развертывает Docker-образ в DockerHub после успешных тестов
3. Использует `DOCKERHUB_USERNAME` и `DOCKERHUB_TOKEN` для аутентификации

### Конфигурация Docker
- Базовый образ: python:3.14-slim
- Порт: 8501 (порт Streamlit по умолчанию)
- Включает эндпоинт проверки состояния
- Оптимизирован для промышленного развертывания

### Развертывание в DockerHub
1. Настройте секреты в репозитории GitHub:
   - `DOCKERHUB_USERNAME`
   - `DOCKERHUB_TOKEN`
2. Запушьте в основную ветку для запуска развертывания
3. Образ помечается как `latest` и с SHA коммита
